# MLhigh v19：彻底重建带箭头坐标轴 + 柱状图贴近坐标轴

## 本轮修复

### 1. 删除旧坐标轴叠加逻辑
上一版是在 Plotly 原生坐标轴基础上追加箭头 annotation，视觉上像“补上去”的箭头，并且多个旧函数链式覆盖后容易重复叠加。

本版处理：
- 最终渲染函数不再调用旧的坐标轴函数链；
- 每次渲染前清理旧的 axis-like shapes 和 annotations；
- 原生 x/y 轴线隐藏；
- 用统一的 shapes + annotations 重建完整带箭头坐标轴。

### 2. 柱状图靠近坐标轴
上一版柱状图仍使用 Plotly 默认分类轴，第一根柱子离 y 轴较远。

本版处理：
- 对分类柱状图，将分类变量映射成连续数值位置；
- 重新设置 tickvals / ticktext；
- x 轴范围设置为 `[-0.62, n-1+0.62]`；
- 第一根柱子会紧贴 y 轴附近，不再大幅远离；
- 柱宽由右侧“柱状图宽度”控制。

### 3. 坐标轴箭头整体化
本版坐标轴不是在原坐标轴末端随便补箭头，而是：
- x 轴线使用 shape 从左到右完整绘制；
- y 轴线使用 shape 从下到上完整绘制；
- x/y 轴箭头使用同一套 annotation；
- 原生 axis showline 关闭，避免双线、错位和“拼接感”。

### 4. 多图标签位置继续固定
- 图形切换标签固定在“结果可视化”顶部；
- 不再被插入居中的 plot 区；
- 如果旧脚本意外把标签放进 plot 容器，新函数会自动移回顶部。

### 5. 已检查
- 前端 charts.js 语法通过；
- app.js 语法通过；
- 后端 Python 编译通过；
- Lasso 用户示例变量组合可返回 5 张图。
