# MLhigh v18：图表渲染失败与图形切换标签位置修复

## 本轮修复重点

### 1. 修复“图表渲染失败”
上一版仍然存在旧图形渲染函数链式覆盖的问题，多个版本的 `polishLayoutForPublication()` 互相包裹，可能把 Plotly 不兼容的 annotation / layout 字段传入浏览器端，导致前端同步报错并显示“图表渲染失败”。

本版修复：
- 新增最终版 `renderCenteredChart()`；
- 不再链式调用旧的高风险 layout polish 函数；
- 统一清洗 Plotly traces 和 layout；
- 移除旧的风险 annotation；
- 使用安全的像素偏移箭头坐标轴；
- 如果 Plotly 因浏览器兼容性异常失败，会自动启用 SVG 安全预览，不再显示“图表渲染失败”。

### 2. 修复图形切换标签跑到中间
上一版图形切换标签被插入到图形预览容器内部，而预览容器本身是居中布局，所以标签会跑到画布中间。

本版修复：
- 将 `chartVariantTabs` 移到结果可视化区域顶部；
- 图形预览区只负责显示当前图；
- CSS 强制结果图形区域使用纵向布局；
- 顶部图形标签固定在图形上方，不再漂浮到中间。

### 3. 保证能出图
本版增加“安全兜底预览”：
- Plotly 正常时，显示完整交互图；
- Plotly 失败时，根据返回数据生成简化 SVG 图；
- 不再用“图表渲染失败”终止展示；
- 确保用户只要分析成功返回图形数据，前端就有可视化反馈。

### 4. 继续保持右侧参数对应逻辑
- 柱状图：柱宽、柱距、柱子颜色；
- 散点图：点大小；
- 折线图：线宽；
- 热图/等高线图：不显示不对应的逐项颜色；
- 字体、背景、透明度、标题继续作用于当前图；
- 图形切换后右侧控件重新匹配当前图。
