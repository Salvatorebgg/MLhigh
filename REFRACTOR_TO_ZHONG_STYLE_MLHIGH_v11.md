# MLhigh v11：结果区合并、jichutongji_zhong 风格对齐与流程加固

## 本轮按反馈修复

### 1. 结果区逻辑重构

左侧步骤栏不再分别放：
- 结果可视化
- 结果表格
- 诊断评估
- 结果讨论

现在左侧只保留一个统一的：
- 5. 结果

右侧结果页顶部提供四个可切换标签：
- 结果可视化
- 表格分析
- 诊断评估
- 结果讨论

运行分析成功后会自动跳转到：
- 结果 → 结果可视化

不会再跳回“分析方法”。

### 2. 图形参数区对齐 jichutongji_zhong

结果可视化页改为：
- 左侧图形预览区
- 右侧图形参数区

右侧图形参数支持：
- 图表主题
- 图表标题
- 坐标轴刻度字体
- 坐标轴标题字体
- 标签字体
- 图形背景：网格线 / 空白 / 透明
- 点 / 标记大小
- 线条宽度
- 柱体间距
- 整体透明度
- 系列颜色逐项调节
- 重置颜色

调整参数后会重新渲染当前图形。

### 3. 表格、诊断、讨论统一放入结果页顶部标签

结果表格、诊断评估、结果讨论不再作为左侧独立步骤，而是整合到结果页顶部标签中，和 jichutongji_zhong 的结果展示逻辑一致。

### 4. 统计流程加固

针对 “Found array with 0 sample(s) while a minimum of 1 is required by NearestNeighbors” 进行处理：

- PSM 校验增加：
  - 处理变量必须是真二分类；
  - 两组至少各有 2 个样本；
  - 至少有 1 个非 ID 协变量；
- PSM 运行阶段增加：
  - 自动把两水平非数值处理变量转换为 0/1；
  - 检查治疗组/对照组是否为空；
  - 防止 NearestNeighbors 空数组报错。

### 5. 运行兜底保护

如果某个方法已经通过后端校验，但底层模型仍遇到边界情况：
- 不再抛出 Internal Server Error；
- 返回稳健执行摘要、描述统计和诊断提示；
- 界面不会卡死或报错中断。

这保证“小白用户只要点到可运行方法，就能得到结果页反馈”。

## 已检查

- 后端 Python 编译通过；

  - 选择三分类 `arm` 作为处理变量时，后端校验为不可运行；
  - 选择二分类 `treatment` 作为处理变量时，后端校验为可运行；
