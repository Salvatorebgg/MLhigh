# MLhigh v8 工作流与验证逻辑修正版

## 修复内容

1. 修复加载综合示例时报错
   - 新增全局 `setStatus()` 兼容函数；
   - 解决“加载失败: setStatus is not defined”。

2. 删除“参数配置”步骤
   - 删除左侧步骤中的“参数配置”；
   - 删除右侧对应页面；
   - 工作流调整为：
     1) 数据上传
     2) 选择变量
     3) 分析方法
     4) 执行分析
     5) 结果可视化
     6) 结果表格
     7) 诊断评估
     8) 结果讨论

3. 变量选择驱动方法判断
   - 不再依赖参数配置页；
   - 根据“研究变量 / 协变量或混杂因素 / 结局变量”自动推导每个方法需要的参数；
   - 能运行的方法显示为可选；
   - 不能运行的方法自动变灰并显示原因。

4. 运行参数自动推导
   - GEE、PSM、生存、马尔可夫、贝叶斯、拉丁方、荟萃、中介、混合效应、复杂抽样、LDSC、机器学习等方法均从变量角色中自动推导参数；
   - 机器学习方法会使用用户选择的研究变量和协变量作为特征，结局变量作为目标变量。

5. 删除左上角品牌文字
   - 删除“MLhigh 高级统计与机器学习”。

6. 检查
   - 前端 JS 语法检查通过；
   - 后端核心 Python 文件编译通过；
   - 说明：本轮未再运行所有重型模型的全量接口压测，避免长时间测试占用；已从前后端逻辑层面对 setStatus、工作流、方法可用性和自动参数推导进行了核对。
