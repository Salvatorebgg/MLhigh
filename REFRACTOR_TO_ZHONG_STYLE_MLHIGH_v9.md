# MLhigh v9：方法可选性与结局类型严格校验修复

## 本轮修复重点

1. 修复分类模型误选连续结局的问题
   - 前端新增严格结局类型判断；
   - 分类模型（逻辑回归、KNN、XGBoost、随机森林、SVM、决策树、CNN、多模型比较）只允许二分类或多分类离散结局；
   - 如果结局是连续型，例如 `outcome_continuous`、连续评分、连续浓度等，分类模型会自动变灰；
   - 连续结局应选择 Lasso/岭回归或对应高级统计模型。

2. 修复 isBinaryVar 过宽的问题
   - 之前只要变量名里含有 `outcome` 就可能被误判为二分类；
   - 现在不再根据变量名粗暴判断；
   - 会结合变量类型、预览数据唯一值数量、是否整数、是否为 0/1 等严格判断。

3. 后端增加兜底保护
   - 即使前端缓存未刷新，用户强行运行分类模型 + 连续目标变量，后端也会返回 400 友好错误；
   - 不再出现 `ValueError: Unknown label type: continuous` 导致 Internal Server Error。

4. 运行参数推导修正
   - 分类模型自动从“结局变量”中选择离散结局；
   - Lasso/岭回归优先选择连续结局；
   - 机器学习特征仍来自“研究变量 + 协变量/混杂因素”。

## 已验证

- 前端 JS 语法检查通过；
- 后端 Python 编译通过；
- 后端保护测试通过：
  - `ml_lr + outcome_continuous` 返回 400 友好提示；
  - `ml_lr + outcome` 返回 200，可正常分析。
